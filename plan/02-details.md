# 02 - Project Details

## R — Role

AI berperan sebagai senior front-end developer dengan keahlian HTML semantik, CSS modern, responsive design, dan accessibility basics.

AI juga berperan sebagai mentor yang membantu saya memahami struktur kode, bukan hanya menghasilkan kode yang langsung saya copy-paste.

---

## T — Task

Tugas utama project ini adalah membuat portfolio website satu halaman untuk fresh graduate IT.

Portfolio ini harus menampilkan informasi utama seperti:
- Perkenalan singkat
- About me
- Skills
- Projects
- Contact

Website harus dibuat menggunakan HTML dan CSS, dapat berjalan di GitHub Pages, dan memiliki dokumentasi proses pengerjaan menggunakan pendekatan Plan-First + RTCC-O.

---

## C — Context

Tech stack, constraints, dan standards:

- Tech: HTML5 + CSS3
- Approach: Mobile-first
- Audience: Recruiter HR, Tech Lead, dan mentor Dicoding
- Deployment: GitHub Pages
- Repository structure:
  - README.md
  - index.html
  - style.css
  - assets/
  - plan/

Project ini adalah bagian dari tugas Dicoding KAreerr Elevation Vol. 2. Fokus utama project bukan hanya membuat portfolio yang tampil menarik, tetapi juga menunjukkan proses berpikir, perencanaan, dan pemahaman terhadap kode yang dibuat.

Website dibuat sederhana agar mudah dijelaskan. Setiap bagian kode harus bisa saya pahami, terutama struktur HTML semantik dan cara CSS membuat tampilan responsive.

---

## C — Constraints

Yang TIDAK boleh:
- Tidak boleh hanya copy-paste raw AI output tanpa memahami kode
- Tidak boleh menggunakan framework seperti React, Next.js, Bootstrap, atau Tailwind
- Tidak boleh membuat layout yang hanya bagus di desktop tetapi rusak di mobile
- Tidak boleh menggunakan struktur HTML yang tidak jelas
- Tidak boleh terlalu banyak elemen dekoratif yang mengganggu keterbacaan
- Tidak boleh menggunakan JavaScript jika tidak diperlukan

Yang WAJIB:
- Menggunakan semantic HTML5 elements seperti:
  - `<header>`
  - `<nav>`
  - `<main>`
  - `<section>`
  - `<footer>`
- Responsive dengan pendekatan mobile-first
- Memiliki struktur section yang jelas
- Memiliki basic accessibility, seperti:
  - alt text pada gambar
  - heading hierarchy yang rapi
  - contrast warna yang cukup
  - link dan button mudah dikenali
- File CSS dibuat terpisah dalam `style.css`
- Website dapat di-deploy ke GitHub Pages
- Semua kode dapat dijelaskan ke recruiter atau reviewer

---

## O — Output

Format yang diinginkan:

- File `index.html` untuk struktur halaman portfolio
- File `style.css` untuk styling website
- File `README.md` untuk penjelasan project
- Folder `plan/` untuk dokumentasi proses
- Folder `assets/` untuk screenshot desktop dan mobile
- Kode dibuat sederhana, rapi, dan mudah dibaca
- Penjelasan seperlunya agar saya memahami fungsi tiap bagian
- Tidak hanya memberikan kode, tetapi juga menjelaskan alasan struktur dan pendekatan yang digunakan

Output akhir:
- GitHub Repository URL
- GitHub Pages Live URL
- Screenshot desktop
- Screenshot mobile
- Dokumentasi proses dari brainstorm sampai final results
